# How to Run Face Subject — Linh An DNA (v2.5)

Face subject mode builds structural identity DNA from multiple images of an AI-generated character.
Unlike room/environment subjects, face DNA has an additional mandatory QC gate to prevent
self-contamination — AI drift baked into DNA → more drift generated → infinite loop.

---

## ⚠ QC Gate 07F — MANDATORY BEFORE ADDING ANY IMAGE

**Only images scoring ≥ 9.0 on rubric 07F may enter the DNA source folder.**

Score every image using `projects/02_KNOWLEDGE/DNA/Linh An Universe/07_LINH_AN_KOL_SYSTEM/07F_QC_CHECKLIST_SCORING_RUBRIC.md`:

| Score | Decision |
|-------|----------|
| ≥ 9.0 | ✅ Approved — add to `data/projects/venho_hotel/media/linh_an/` |
| 8.0–8.9 | ⚠ Conditional — use as reference only, NOT as DNA source |
| < 8.0 | ❌ Reject — do not add |

`data/projects/venho_hotel/media/linh_an/` is a **curated selection**, not a dump of all generated images.

### Why the gate exists

Linh An is an AI-generated character. Her images are not photographs of a real person — they are
generated by a model that can drift. If a drifted image enters the DNA source:

```
Drifted image enters DNA source
       ↓
DNA captures the drift as "identity"
       ↓
DNA generates prompts with drift baked in
       ↓
New images drift further
       ↓
Self-contamination loop — identity lost
```

The QC gate breaks this loop by keeping only high-fidelity images in the source pool.

## Running Face DNA

```bash
# Add QC-passing images to the source folder first
cp <approved_image.png> data/projects/venho_hotel/media/linh_an/

# Run Mode B
venho vision observe --mode b --project venho_hotel --subject linh_an \
  --input data/projects/venho_hotel/media/linh_an

# Or via --all (will include linh_an if folder is non-empty)
venho vision observe --all --project venho_hotel
```

## What Face DNA Contains

### INVARIANT — Structural Identity (must be stable)

These keys make Linh An recognizable across all images:

| Key | What it captures |
|-----|-----------------|
| `face_shape` | elongated oval, balanced proportions |
| `eye_shape` | long almond, horizontal emphasis |
| `eyebrow_position` | low, close to eye |
| `nose_bridge` | slim natural |
| `lip_shape` | natural full, slightly thinner upper |
| `lip_corners` | very subtle upward |
| `jaw_line` | soft feminine |
| `chin_shape` | delicate pointed |
| `skin_tone` | fair warm ivory |
| `hair_color` | dark chocolate brown |
| `hair_length` | long past shoulders |
| `earring_type` | small pearl drop (present in EVERY image) |

### VARIABLE — Appearance (allowed to vary)

These keys change across shoots and are intentionally variable:

| Key | Expected values |
|-----|----------------|
| `camera_angle_horizontal` | soft_hero_left, straight_on, profile_left/right |
| `expression_type` | living_expression_subtle_smile, neutral |
| `eye_opening` | slightly_narrow, standard |
| `hairstyle` | elegant_low_bun, long_soft_waves, natural_loose |
| `eye_color` | warm_brown, dark_brown (lighting-dependent) |

> **Note on hairstyle:** If source images all show the same hairstyle, it will be classified as INVARIANT
> by coverage/consistency rules. This is statistically correct — it reflects what the source pool
> actually shows, not a bug. Add images with different hairstyles to make it VARIABLE.

## Critical Rules for Face Subjects

1. **Grounding is permanently disabled** — the observe prompt forbids web search and identity
   recognition. This is not configurable. Confirmed in `prompts/observe_face.md`.

2. **Do not identify real people** — observe only structural characteristics. No name, no social
   media handle, no public identity references.

3. **One character = one subject** — never mix images of different characters or different people
   into the same face DNA.

4. **Overlay is the primary source of FORBIDDEN** — curated rules in `linh_an.overrides.yaml`
   cover drift types the model cannot observe (e.g., "no plastic skin," "no generic influencer look").
   Do not rely on machine observation for FORBIDDEN.

---

## Curated Overlay (`linh_an.overrides.yaml`)

Located at `config/projects/venho_hotel/subjects/linh_an.overrides.yaml`.

Key contents:
- `forbidden:` — 9 curated drift-prevention rules (no plastic skin, no generic Asian face, etc.)
- `allowed_imperfections:` — 4 curated natural variation rules (eye asymmetry, lip corner unevenness)
- `wording_overrides:` — replaces machine wording for structural keys with v3.1 canonical descriptions
- `notes:` — identity stack reference, QC threshold, signature accessories

Editing the overlay does not trigger vision API calls or bump `dna_version`. Run Mode B again to re-render.

## Adding More Images Later

1. Score the new image with rubric 07F — must be ≥ 9.0.
2. Copy to `data/projects/venho_hotel/media/linh_an/`.
3. Run Mode B — old DNA archived automatically, new version generated.

If images are unchanged, Mode B detects "no change" and skips re-generation.
